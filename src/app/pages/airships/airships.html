✈ Modding Airships: Conquer the Skies

Using Sail — a Fabric-based modloader for Airships

What is Sail?

Sail is a custom Fabric game provider that brings the Fabric modding ecosystem to Airships: Conquer the Skies. It lets you write mods in Java using the same tools as Minecraft modding — Fabric Loader, Mixins, and the standard ModInitializer API.

Under the hood, Sail implements GameProvider so Fabric's Knot launcher can inject bytecode hooks into the game before its main class even runs. Two built-in patches are always applied: one injects the mod initialization lifecycle, and another shows "Modded Fabric" on the main menu.

Installation

  1. Download the latest Sail release (the sail/ folder from the GitHub release).
  2. Place the following files next to your game JAR:
    • AirshipsGameProvider-1.0.0.jar
    • fabric-dependencies/ — all Fabric runtime JARs
    • mods/ — drop your mod JARs here
  3. Launch the game with Knot instead of the default entry point: 
    java -Dhttps.protocols=TLSv1.2,TLSv1.1,TLSv1 \
  -Dfabric.skipMcProvider=true \
  -Dfabric.side=client \
  -cp "game.jar:AirshipsGameProvider-1.0.0.jar:fabric-dependencies/*" \
  net.fabricmc.loader.impl.launch.knot.KnotClient

Creating a Mod

A Sail mod is a standard Fabric mod with a Gradle project. The minimal setup:

1. fabric.mod.json

Every mod needs a manifest at src/main/resources/fabric.mod.json:

{
  "schemaVersion": 1,
  "id": "my-mod",
  "version": "1.0.0",
  "name": "My Mod",
  "description": "Does something cool in Airships",
  "entrypoints": {
    "main": ["com.example.MyMod"]
  }
}

2. Mod initializer

Implement ModInitializer — this runs once when the game loads your mod:

public class MyMod implements ModInitializer {
    @Override
    public void onInitialize() {
        System.out.println("My mod loaded!");
    }
}

3. build.gradle dependency

repositories {
    maven { url "https://maven.fabricmc.net/" }
    maven { url "https://jitpack.io" }
}

dependencies {
    api "com.github.KenroLe:sail:1.0.0"
}

Demo Mod

The simplest working mod — it logs all loaded mods to the console on startup. A good way to verify that Sail is set up correctly. 

public class DemoMod implements ModInitializer {
    @Override
    public void onInitialize() {
        Log.info(LogCategory.GENERAL, "Demo Mod loaded! Airships Fabric works!");
        FabricLoader.getInstance().getAllMods().forEach(mod ->
            Log.info(LogCategory.GENERAL,
                "  - " + mod.getMetadata().getId()
                + " " + mod.getMetadata().getVersion())
        );
    }
}

Example: Injured Modules Mod

A real mod that makes damaged lift modules provide less lift proportional to how much HP they have lost. It uses a Mixin to inject code into the game's Airship.availableLift() method without touching the original source.

mixins.json

{
  "required": true,
  "minVersion": "0.8",
  "package": "com.kenro.injuredmodulesmod.mixin",
  "compatibilityLevel": "JAVA_8",
  "mixins": ["HpAffectsLiftMixin"]
}

HpAffectsLiftMixin.java

@Mixin(Airship.class)
public abstract class HpAffectsLiftMixin extends GridBody {

    @Shadow public ArrayList<Module> modules;
    @Shadow public final BonusSet currentBonuses = null;

    @Inject(
        at = @At(value = "INVOKE", ordinal = 1, target = "size", shift = At.Shift.AFTER),
        method = "availableLift(Lcom/zarkonnen/airships/Combat;)I"
    )
    public void availableLift(Combat c, CallbackInfoReturnable ci,
            @Local(ordinal = 0) LocalIntRef result) {

        for (Module m : modules) {
            if (m.type.hasLift() && m.canRun()) {
                float damageRatio = 1 - (float) m.hp / m.maxHP;
                int liftLoss = (int) (m.type.getLift(currentBonuses) * damageRatio);
                result.set(result.get() - liftLoss);
            }
        }
    }
}

The @Mixin targets Airship, @Inject picks the exact bytecode location inside availableLift, and @Local captures the local variable holding the running lift total so we can reduce it in-place.

How Mixins Work

Mixin is a bytecode-weaving framework that lets you inject code into game classes at load time — no decompiling or patching JARs needed. Common injection points:

AnnotationWhat it does
@InjectInject a callback at a specific point in a method
@OverwriteReplace a method entirely
@RedirectReplace a specific method call inside a method
@ModifyVariableModify a local variable at a specific point
@ShadowReference a field/method from the target class
